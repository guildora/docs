# i18n

Guildora apps must support at least English (`en`) and German (`de`).

## File Structure

```
src/i18n/
  en.json    ← English (required)
  de.json    ← German (required)
```

## Translation Files

```json
// src/i18n/en.json
{
  "app": {
    "title": "My App",
    "welcome": "Welcome back, {username}!",
    "overview": {
      "heading": "Overview",
      "membersTracked": "Members tracked"
    }
  },
  "error": {
    "load": "Failed to load data."
  }
}
```

```json
// src/i18n/de.json
{
  "app": {
    "title": "Meine App",
    "welcome": "Willkommen zurück, {username}!",
    "overview": {
      "heading": "Übersicht",
      "membersTracked": "Erfasste Mitglieder"
    }
  },
  "error": {
    "load": "Daten konnten nicht geladen werden."
  }
}
```

## Usage in Vue Pages

No import needed — `useI18n()` is globally injected:

```typescript
const { t } = useI18n()

// Simple key
t('app.title')  // → "My App"

// With interpolation
t('app.welcome', { username: 'Alice' })  // → "Welcome back, Alice!"
```

The host automatically selects the language based on the user's locale preference.

## Rules

- Both `en.json` and `de.json` must exist and have identical key structure.
- Every new UI text — labels, buttons, hints, status messages, ARIA texts — must go through the translation system.
- Keys must be in camelCase, nested logically.
- Never hardcode UI text directly in Vue templates.
